## 📋 Overview

<a href="#fontExplorerX">FontExplorer X</a> is a professional font management application for Mac — providing the tools that designers, typographers, and creative studios need to organize, activate, preview, and intelligently manage large font libraries without the performance and conflict problems that arise from installing all fonts simultaneously.

The <a href="#fontExplorerX">font activation and deactivation system</a> is the core of FontExplorer X's value — fonts load into the system only when needed rather than all being permanently installed. A designer with thousands of fonts doesn't run all of them simultaneously, slowing every application's font menu and risking conflicts between fonts with overlapping names or metrics. FontExplorer X activates specific fonts for specific projects and deactivates them when the project is complete, keeping the active font set small and the system performing well. The <a href="#fontExplorerX">sets and smart sets organization</a> creates named collections of fonts — organizing by project, client, typeface classification, purchase source, or any organizational principle the library requires. Smart sets use criteria-based rules to automatically populate font collections — all fonts from a specific foundry, all serif fonts, all fonts added in the last month — without manual curation as the library grows.

The <a href="#fontExplorerX">font preview system</a> renders any font in the library at any size with custom sample text — visually comparing fonts before choosing one for a project, identifying fonts by their visual characteristics, and browsing the library as a visual catalog rather than an alphabetical list of names. The <a href="#fontExplorerX">type classification system</a> organizes fonts by category — serif, sans-serif, script, display, monospace, symbol — using the classification metadata embedded in font files and augmented by FontExplorer X's own classification tools.</p>

<a href="#fontExplorerX">Font conflict detection</a> identifies fonts with conflicting names or metrics — the technical conflicts that cause unexpected font substitution, incorrect rendering, and document reflow when multiple fonts claim the same name. FontExplorer X detects these conflicts before they cause problems in design documents. <a href="#fontExplorerX">Adobe Creative Suite auto-activation</a> detects when a document requires a specific font that isn't currently active and activates it automatically — opening an InDesign or Illustrator document that uses an inactive font activates that font without manual intervention, preventing the missing-font substitution that breaks document appearance.

<a href="#fontExplorerX">Font metadata and information</a> displays detailed technical information about each font — PostScript name, supported Unicode ranges, OpenType feature support, character coverage, licensing information embedded in the font file — for making informed font selection decisions. <a href="#fontExplorerX">Font comparison views</a> display multiple fonts simultaneously with the same sample text for side-by-side visual comparison across the library.</p>

## 🤔 FAQ

<details>
<summary>Why not just use Font Book?</summary>

Font Book is adequate for small font libraries. FontExplorer X provides project-based sets, Adobe auto-activation, conflict detection, visual preview, and organization tools that Font Book doesn't offer, making it significantly more capable for large libraries and professional workflows.

</details>

<details>
<summary>What is font auto-activation?</summary>

Auto-activation detects when an InDesign, Illustrator, or other Adobe application document requires a specific font and activates it automatically, preventing missing-font substitution without manual intervention.

</details>

<details>
<summary>Does FontExplorer X work with all fonts?</summary>

FontExplorer X works with OpenType, TrueType, and PostScript Type 1 fonts — the formats used by professionally produced fonts.

</details>

<details>
<summary>What are Smart Sets?</summary>

Smart Sets are automatically populating font collections based on criteria — all fonts from a specific foundry, all serif fonts, all fonts added recently — without manual curation as the library grows.

</details>

<details>
<summary>Does FontExplorer X detect font conflicts?</summary>

Yes. Conflict detection identifies fonts with the same PostScript name or conflicting metrics that would cause font substitution and rendering problems.

</details>

<details>
<summary>Does FontExplorer X integrate with Photoshop?</summary>

Yes. Auto-activation plugins work with InDesign, Illustrator, and Photoshop in Adobe Creative Suite.

</details>

<details>
<summary>Can FontExplorer X manage fonts not in the system Fonts folder?</summary>

Yes. Fonts import from any location and are managed through FontExplorer X's library without requiring them to be in the system Fonts folder.

</details>

<details>
<summary>Does FontExplorer X support Apple Silicon?</summary>

Yes. Current versions support Apple Silicon Macs natively.

</details>

<details>
<summary>Can I compare fonts side by side?</summary>

Yes. Font comparison views display multiple fonts with the same sample text simultaneously for visual comparison.

</details>
